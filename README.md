# Двусвязный список с поддержкой произвольных ссылок (Random Ptr)

## Описание проекта

Данный проект реализует двусвязный список (`DoubleLinkedList`) с расширенной функциональностью для работы с указателями на произвольные элементы (`rand`). Основная задача — корректная сериализация (сохранение в файл) и десериализация (восстановление из файла) сложной структуры данных, где элементы могут ссылаться не только на соседей, но и на случайные узлы в пределах списка.

## Формат входных данных файла inlet.in.
Простой текстовый формат. Каждая строка описывает один узел:
`<data>`;`<rand_index>` 
Где:
- `<data>` — строка (могут быть пробелы, спецсимволы, кодировка UTF-8)
- `<rand_index>`— индекс узла, на который указывает rand, либо -1, если rand == nullptr
Пример входного файла (inlet.in):
apple;2
banana;-1
carrot;1
Этот список:
- Узел 0: "apple" → rand на узел 2
- Узел 1: "banana" → rand = nullptr
- Узел 2: "carrot" → rand на узел 1

## Сборка и запуск

1. Создаем отдельную директорию для сборки
mkdir build && cd build

2. Конфигурируем проект
cmake ..

3. Собираем исполняемый файл
cmake --build .

4. Запускаем программу
./bin/double_linked_list ../resources/inlet.in ../resources/outlet.out

## Версии
Для решения поставленной задачи предствалено 2 версии. Каждая характерна по своему.

# Реализация version_1
Наиболее примечательной частью является класс `RandIndexResolver`.
*   **Проблема**: При десериализации узел может ссылаться через `rand_index` на элемент, который еще не был создан (находится дальше в файле).
*   **Решение**: Класс работает как **двухпроходный резолвер "на лету"**.
    *   **Регистрация**: Каждому созданному узлу присваивается индекс.
    *   **Прямая ссылка**: Если `rand_index` меньше текущего, ссылка устанавливается сразу.
    *   **Отложенная ссылка**: Если `rand_index` больше текущего, ссылка помещается в `pending_rands_` (словарь ожидания).
    *   **Разрешение**: Как только создается узел, соответствующий ожидаемому индексу, все ноды, ожидавшие его, мгновенно получают корректный указатель `rand` через `std::forward_list`.
*   **Эффективность**: Восстановление всех связей выполняется за **O(N)** без дополнительных проходов по списку после полной загрузки.

# Реализация version_2
Ключевым улучшением является оптимизированный класс `DataSerializer`.

**Особенности**:
- `DataSerializer` создаёт узлы списка, но **не владеет** ими. Ответственность за освобождение памяти лежит на вызывающем коде. Для очистки используется функция `void ClearList(ListNode* head)`.
- **Сложность десериализации**: **O(n)**
  - Построение списка: O(n)
  - Установка `rand`-связей: O(n)
- **Сложность сериализации**: **O(n)**
- Линейная сложность достигается за счёт использования `std::unordered_map`, что обеспечивает константное время доступа к узлам по индексу ценой дополнительных накладных расходов по памяти.
